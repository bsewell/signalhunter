The Process

Not a methodology.
A working system.

This isn't a consulting framework or a slide deck. It's the actual infrastructure we use to build products — running in production right now.

The Virtual Board of Advisors

22 people who never attend a meeting.

Every advisor has a JSONL file. Every entry is sourced — a real quote, a documented principle, a recorded interview. The AI reads them before making decisions. This is indexed institutional wisdom, not decoration.

Institutional Memory

Nothing gets forgotten. Nothing gets repeated.

Traditional teams lose context every time someone goes on vacation, changes roles, or just forgets a Slack thread from three months ago. We solved this with three append-only files.

decisions.jsonl

Every significant choice: what we decided, what we rejected, why. When the AI starts a new session, it reads this first. It will never re-propose something we already tried and abandoned.

failures.jsonl

Every broken assumption, with root cause and prevention. Example: the InsightEngine checked channel === 'CH2' but the database stored it as a number. Filtering returned zero results. Documented once, never happens again.

experiences.jsonl

Moments worth remembering. The first successful build. The insight that changed the architecture. Context that gives future sessions the judgment a new team member would take months to develop.

A Build Session

What it actually looks like day-to-day.

Session Start

The AI checks which git worktree it's in, which one Xcode is building from, reads the last session state, reads the last three failures. It announces all of this before touching a single file. This protocol exists because we once wasted hours editing the wrong worktree.

The Operator Decides

You describe what you want in plain language. "I want users to log a meal by taking a photo and have AI identify the ingredients." No PRDs. No Jira tickets. The AI debates feasibility, checks the board for relevant principles, and writes the code.

Architecture Is Protected

Permanent guardrails prevent architectural drift. All writes go through the EventRepository protocol. No direct database imports in views. Swapping from Google Sheets to Supabase is a one-line change. This is intentional vendor lock-in prevention.

Decisions Get Logged

After every significant choice, it's appended to the decision log. After every failure, root cause is documented. After every session, a machine-readable checkpoint is written. The next session picks up exactly where this one left off.

This is a real system, running in production.
